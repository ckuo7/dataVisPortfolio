Dataset Summary

My project works on two bikeshare datasets. The first one is trip history where each row contains starting station, ending station, starting time, ending time and duration of a trip. To visualize net bikes number (number of bikes coming in minus number of bikes coming out) of a given station in a particular time, I aggregated this dataset by station names and hour to obtain the number. In addition, to be able to plot each station on map, I included geolocation information for each station. The final dataset has following columns:

  • station: name of the station
  • hour
  • output: number of bikes coming out
  • input: number of bikes coming in
  • net: input minus output
  • lon: longitude of the station
  • lad: latitude of the station
  • zip code
  • geo information: other geolocation information.

The second dataset includes extraneous variables such as temperature, wind speed, and humidity

Sketch Summary

The whole purpose of this data visualization project is to understand the distribution of bikes for a given time. To achieve this goal, I decided to create a dashboard with three separate visualization. The first one is a two-sided bar chart that shows the net number of bikes for each station at a given hour. The second one is a line chart to display my time-series variables such as temperature and humidity. The last one is a map that indicates the location of each station.
