Planned Tools

  • Data Preparation: Python, R
  • Data Visualization: D3, TopoJSON
  • Prototype: Tableau
  • API: Google geo-location API

Planned Techniques

  • Line Chart: My first dataset contains time series data of temperature, humidity and number of bikes rented, I planed to implement a line chart to visualize variables trends over time. Hence, time stamps would be columns and other variables would be rows.
  • Bubble Plot: The data is a mixture of categorical and continuous variables. So, a bubble chart would be a good choice here to encode continuous variables along X and Y axes, and also the area of bubbles; whereas color and opacity can be used to encode categorical information.
  • Symbol Map: The second dataset contains number of bikes rented and returned at given location and time, which makes it a good resource to implement geographical visualization. I am thinking of using longitude and latitude to plot the location of each station on the map and perhaps encoding the net bike numbers as the area of circles. Or, plot a simple map and use a barchart to visualize bikes number of stations.
  • Bar chart: For my second dataset. I also plan to aggregate number of bikes over time and plot a barchart with station names as y axis and sum of number of bikes as x axis. I think barchart makes it very easy and simple to compare the number between stations.

Planned Interaction

  • I plan to implement focus and context on my line chart with filter functionality so that I can visualize all the time series variables at one place.
  • For map chart, zooming and dragging are potential candidates.
  • For bar-chart, I plan to implement some detail on demand like tooltips to show exact bike number of a station and perhaps linked views on maps. Specifically, region of where a station belongs to would change its color or opacity on map chart to indicate geo-location information.
